# go-lab

Personal Go workspace for building projects, solving exercises, and keeping a small knowledge base of references worth revisiting later.

This repository is not only a collection of standalone Go projects. It also documents the learning process:

- `solutions/` contains Go practice exercises and learning solutions.
- `knowledge-base/` is reserved for notes, references, and reminders I want to keep around as I learn and come back to over time.
- The project directories at the root are independent experiments, apps, or playgrounds.

## Repository Structure

- `go-serverless` - Serverless user API on AWS using Lambda, API Gateway, and DynamoDB.
- `pokedexcli` - Pokemon-themed CLI with cached PokeAPI requests and simple map navigation.
- `slot-machine` - Terminal slot machine game with betting and payout logic.
- `ecommerce-project` - Ecommerce-focused Go application workspace.
- `go-routines` - Small playground for concurrency and goroutine-related examples.
- `lab` - Scratch area for quick Go experiments and tests.
- `solutions/` - Go learning exercises and problem solutions.
- `knowledge-base/` - Personal reference material for concepts, patterns, and things to remember later.

## Purpose

The goal of this repo is to keep projects and learning material in one place:

- Build small to medium Go projects.
- Practice Go through exercises in `solutions/`.
- Keep a reference-friendly knowledge base for topics I may forget and need again later.

## Notes

- Kubernetes-related content has been removed from this repository.
- Each project can evolve independently and may have its own README and setup steps.
